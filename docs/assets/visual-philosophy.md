# Midnight Scaffold

`OpenClaw Docker` 的视觉表达不应该像一份普通部署文档，而应该像一张“把复杂系统整理清楚”的控制台海报。它的核心不是炫技，而是把工程感、秩序感和可控感变成第一眼就能读懂的视觉印象：深色背景像机房与终端，明亮的蓝青色像稳定在线的服务状态，少量暖色只在关键节点出现，告诉读者这里既有系统能力，也有人工设计过的运维体验。

空间上，这套视觉语言强调“稳定支撑 + 清晰分层”。大块留白不是空，而是为了让服务边界、数据边界和控制边界显得更明确。每个模块都应当像经过仔细排布的控制面板，彼此之间关系明确、节奏克制、层次分明，让读者自然感觉这是一个被精心整理过、可以长期维护的工程系统。整个画面必须呈现出被反复推敲过的秩序感，像一位经验很深的工程师亲手把混乱熨平之后留下的结构。

颜色和材质要服务于“可信赖的工程气质”。主色以深海军蓝、墨黑、青蓝渐变为底，辅以少量高亮蓝、青绿和琥珀色强调状态、入口和关键数据。材质感应当接近高质量产品发布图：柔和发光、干净描边、精细网格、低噪点渐变，而不是廉价霓虹或花哨特效。每个色块、阴影、圆角和线条都要像经过无数轮微调，体现出极高的完成度与克制感。

构图上，信息表达应当更多依赖模块关系、连线、大小、对比和空间节奏，而不是长篇文字。文字只作为视觉锚点：一句主标题、几个能力标签、少量节点名和说明词即可。最终作品必须看起来像花了大量时间精修过的发布素材——对齐、节奏、间距、对比、层次都必须有“高手做过”的感觉，给人一种这不是临时拼出来的部署仓库，而是一套认真经营过的基础设施产品。
