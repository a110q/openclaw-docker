架构总览 / Architecture Overview

把配置、运行时、后台和宿主机能力拆清楚，用户更容易部署，维护者也更容易继续迭代。

用户入口

  • `18789`：Control UI / Gateway
  • `18889`：平台入口 + 运维后台
  • `18080`：ClawSwarm 协作运行时

配置边界

  • `.env` 负责部署层变量
  • `openclaw.json` 负责 Gateway 运行配置
  • 宿主机数据目录负责状态与工作区
宿主机数据根目录配置、日志、工作区、缓存、后台状态统一落地，便于备份、迁移和排障OPENCLAW_HOST_DATA_ROOT
openclaw-gateway主服务，负责 Control UI、模型调用、Agent 生命周期和 sandbox 生命周期。18789 / 18790
openclaw-admin-ui后台控制台，负责服务控制、Provider 管理、Agent 纳管和状态查看。18889
openclaw-clawswarm协作运行时底座，为任务、消息和 OpenClaws 扩展预留能力边界。18080
host capability bridge通过插件把宿主机动作纳入受控通道，而不是直接取消 sandbox。plugins/openclaw-host-ops
openclaw-mysql平台控制面数据库，保存用户、会话、Provider 和运行态映射。23306

为什么这样拆

  • 新手部署路径更明确
  • 数据、配置、运行时边界更清楚
  • 适合继续加平台能力而不把主链路改乱

关键价值

部署更稳定
运维更可见
配置更可解释
扩展更有边界
